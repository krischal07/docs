Use the reviewed Postman collection for hands-on merchant lifecycle, partner API, webhook, and customer-verification testing without changing the underlying request definitions.

## Import Instructions

<Steps>
  <Step title="Download The JSON File">
    Save the raw `postman-collection.json` file directly from this documentation portal.
  </Step>
  <Step title="Import Into Postman">
    In Postman, choose import and select the unchanged collection file.
  </Step>
  <Step title="Fill Environment Values">
    Set `baseUrl`, `merchantToken`, `partnerKey`, `storeId`, `outletId`, `integrationKey`, `expectedRestaurantId`, `expectedRestaurantName`, `customerPhone`, `customerEmail`, and `token`.
  </Step>
</Steps>

## Recommended Testing Order

<Steps>
  <Step title="Create And Connect">
    Create the integration, connect the restaurant, and confirm the integration reaches `CONNECTED`.
  </Step>
  <Step title="Submit Test Sale">
    Send the first valid sale and confirm the integration reaches `ACTIVE`.
  </Step>
  <Step title="Verify Customer Outcome">
    Search for the customer, fetch customer details, and verify loyalty data and points were populated.
  </Step>
  <Step title="Verify Failure Cases">
    Send the mismatch webhook and confirm the integration health moves to `ERROR` without changing the active contract.
  </Step>
  <Step title="Exercise Lifecycle">
    Run disconnect, reconnect, and rebind checks, then use the verification folder to confirm the expected states.
  </Step>
</Steps>

<Info>
The collection only uses the active singular connect contract. No request in the active collection uses the legacy `account + locations[]` payload.
</Info>
