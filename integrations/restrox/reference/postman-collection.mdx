## Import Instructions

<Steps>
  <Step title="Download The JSON File">
    Save the raw `postman-collection.json` file directly from this documentation portal.
  </Step>
  <Step title="Import Into Postman">
    In Postman, choose import and select the unchanged collection file.
  </Step>
  <Step title="Fill Environment Values">
    Set `baseUrl`, `partnerKey`, `integrationKey`, `expectedRestaurantId`, `expectedRestaurantName`, `customerPhone`, and `token`.
  </Step>
</Steps>

## Collection Sections

- `Partner API`
- `Webhook Verification`
- `Customer API`

## Recommended Testing Order

<Steps>
  <Step title="Connect Restaurant">
    Connect the restaurant using your integration key and confirm the response returns `CONNECTED` and a webhook token.
  </Step>
  <Step title="Submit Test Sale">
    Send the first valid sale and confirm the integration reaches `ACTIVE`.
  </Step>
  <Step title="Verify Customer Outcome">
    Search for the customer, fetch customer details, and verify loyalty data and points were populated.
  </Step>
  <Step title="Verify Failure Cases">
    Confirm invalid webhook token handling, disconnected integration state, and missing restaurant binding before connect.
  </Step>
</Steps>

<Info>
The collection uses the active singular connect contract and phone-first webhook examples. RestroX customer search uses `x-partner-key` plus `x-integration-key` against `/api/partners/restrox/customers/search`.
</Info>
