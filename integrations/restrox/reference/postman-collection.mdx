# Postman Collection

Use the reviewed Postman collection for hands-on connect and webhook testing without changing the underlying request definitions.

<Columns cols={2}>
  <Card title="Download Collection" icon="download" href="/postman-collection.json">
    Download the canonical `postman-collection.json` file used for partner testing.
  </Card>
  <Card title="Testing Guide" icon="flask-conical" href="/testing-guide">
    Pair the collection with the documented response expectations and fixture order.
  </Card>
</Columns>

## Import Instructions

<Steps>
  <Step title="Download The JSON File">
    Save the raw `postman-collection.json` file directly from this documentation portal.
  </Step>
  <Step title="Import Into Postman">
    In Postman, choose import and select the unchanged collection file.
  </Step>
  <Step title="Fill Environment Values">
    Set `baseUrl`, `partnerKey`, `integrationKey`, `expectedRestaurantId`, `expectedRestaurantName`, and `token`.
  </Step>
</Steps>

## Recommended Testing Order

<Steps>
  <Step title="Connect">
    Bind the restaurant first and confirm the response status is `CONNECTED`.
  </Step>
  <Step title="Completed Sale">
    Confirm the first happy-path `order.completed` delivery is accepted.
  </Step>
  <Step title="Duplicate Replay">
    Re-send the same sale request once to confirm duplicate-safe behavior.
  </Step>
  <Step title="Refund">
    Send the refund request that references the original sale identifier.
  </Step>
  <Step title="Failure Cases">
    Validate legacy connect rejection, invalid token, and wrong-location behavior before go-live.
  </Step>
</Steps>

<Info>
The collection includes the active singular connect contract. The legacy `account + locations[]` payload is intentionally not included.
</Info>
