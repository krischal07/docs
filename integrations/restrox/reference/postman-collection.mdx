## Import Instructions

<Steps>
  <Step title="Download The JSON File">
    Download the raw `postman-collection.json` file directly from this documentation portal. The link above forces a file download.
  </Step>
  <Step title="Import Into Postman">
    In Postman, choose import and select the unchanged collection file.
  </Step>
  <Step title="Fill Environment Values">
    Set the webhook base URL, token, and any request variables required by your testing environment.
  </Step>
</Steps>

## Recommended Testing Order

<Steps>
  <Step title="Completed Sale">
    Confirm the first happy-path `order.completed` delivery is accepted.
  </Step>
  <Step title="Duplicate Replay">
    Re-send the same sale request once to confirm duplicate-safe behavior.
  </Step>
  <Step title="Refund">
    Send the refund request that references the original sale identifier.
  </Step>
  <Step title="Failure Cases">
    Validate invalid token and wrong-location behavior before go-live.
  </Step>
</Steps>

<Info>
The collection includes the current RestroX-native field names and store-level mapping examples.
