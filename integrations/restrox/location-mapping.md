Not applicable for RestroX outlet-owned architecture.

This page is preserved so older links continue to resolve. RestroX does not use:

- `PosIntegrationLocation`
- location mapping
- review mapping
- location sync
- location approval
- draft bootstrap location records as part of the active RestroX contract
- location webhook token routing

Current RestroX bindings are stored directly on `PosIntegration.external_location_id` and `external_location_name`.
