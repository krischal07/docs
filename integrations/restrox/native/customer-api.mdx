# Customer API

Samparka exposes read-only partner customer APIs for RestroX. Partners can search and retrieve customers, but customer creation is not exposed through partner APIs.

## Purpose

Use this page to implement:

- customer search
- customer retrieval

## Concepts

All customer partner routes require:

- partner authentication
- `x-integration-key`
- provider path parameter

The current implementation supports provider `restrox`.

Customers are managed by Samparka.

## Authentication

Required headers:

```http
x-partner-key: your-partner-key
x-integration-key: SPK-RX-ABC12345
```

You can also send the partner key as:

```http
Authorization: Bearer your-partner-key
```

## Customer Identity Notes

These endpoints depend on the identity outcomes documented in [Customer Identity](./customer-identity):

- `found`
- `missing`
- `conflict`
- `invalid_phone`

## Endpoints

### Search Customer

```http
GET /api/partners/{provider}/customers/search?phone=9801234567
```

Supported query:

- `phone`

No other search keys are accepted by the verified implementation.

Example success response when found:

```json
{
  "exists": true,
  "customer": {
    "id": "687000000000000000000001",
    "name": "Asha",
    "phone": "9801234567",
    "email": "asha@example.com",
    "points": 45,
    "tier": "Bronze Tier",
    "lifetimePoints": 45,
    "membershipSince": "2026-06-08T10:15:00.000Z"
  }
}
```

Example success response when missing:

```json
{
  "exists": false
}
```

Failure cases:

- `400 Phone is required`
- `400 Search supports exact phone lookup only`
- `400 Invalid phone number`
- `409 customer_identity_conflict`

### Get Customer

```http
GET /api/partners/{provider}/customers/{customerId}
```

Example response:

```json
{
  "customer": {
    "id": "687000000000000000000001",
    "name": "Asha",
    "phone": "9801234567",
    "email": "asha@example.com",
    "points": 45,
    "tier": "Bronze Tier",
    "lifetimePoints": 45,
    "membershipSince": "2026-06-08T10:15:00.000Z"
  }
}
```

Failure cases:

- `404 Customer not found`

## Operational Notes

- Customer APIs are read-only in the current backend implementation.
- Customer creation is not exposed through partner APIs.

## Troubleshooting Notes

- If search fails with `Invalid phone number`, normalize and validate the phone before retrying.
- If search returns conflict, manual customer cleanup is required before automated partner flows can rely on that phone identity.
- If get-by-id returns `Customer not found`, confirm the customer belongs to the store resolved by the supplied `x-integration-key`.
