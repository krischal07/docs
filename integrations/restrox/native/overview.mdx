RestroX uses Samparka's outlet-owned POS model.

## In Scope

- merchant creates one integration per outlet
- Samparka issues one Integration Key and one webhook token per integration
- RestroX binds exactly one restaurant to the outlet
- webhook delivery activates the integration after the first valid sale

## Core Invariant

```text
provider = restrox
ownership_mode = OUTLET_OWNED
```

## Stored Restaurant Binding

Restaurant identity is stored directly on `PosIntegration`:

- `external_location_id`
- `external_location_name`

## Lifecycle And Health

Integration lifecycle:

```text
CREATED -> CONNECTED -> ACTIVE -> DISCONNECTED
```

Health state:

```text
HEALTHY | WARNING | ERROR
```

## What Changes Status

- create integration -> `CREATED`
- partner connect -> `CONNECTED`
- first valid `sale.completed` webhook -> `ACTIVE`
- merchant disconnect -> `DISCONNECTED`
- restaurant mismatch webhook -> health moves to `ERROR`
