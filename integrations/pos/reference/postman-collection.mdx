# Postman Collection

Use the Postman collection for hands-on partner API, webhook, and customer testing without changing the underlying request definitions.

## Import Instructions

<Steps>
  <Step title="Download The JSON File">
    Save the raw `postman-collection.json` file directly from this documentation portal.
  </Step>
  <Step title="Import Into Postman">
    In Postman, choose import and select the unchanged collection file.
  </Step>
  <Step title="Fill Environment Values">
    Set `baseUrl`, `providerApiKey`, `integrationKey`, `expectedLocationId`, `expectedLocationName`, `customerPhone`, and `webhookToken`.
  </Step>
</Steps>

## Collection Sections

- `Partner API`
- `Webhook Verification`
- `Customer API`

## Recommended Testing Order

<Steps>
  <Step title="Connect Location">
    Connect the external location using your integration key and confirm the response returns `CONNECTED` and a webhook token.
  </Step>
  <Step title="Submit Test Sale">
    Send the first valid sale and confirm the integration reaches `ACTIVE`.
  </Step>
  <Step title="Verify Customer Outcome">
    Search for the customer, fetch customer details, and verify loyalty data and points were populated.
  </Step>
  <Step title="Verify Failure Cases">
    Confirm invalid webhook token handling, disconnected integration state, and missing location binding before connect.
  </Step>
</Steps>

<Info>
The collection uses the preferred location connect contract and phone-first webhook examples. Partner API requests use `Authorization: Bearer {{providerApiKey}}`, and POS customer search also includes `x-integration-key` against `/api/partners/{provider}/customers/search`.
</Info>
